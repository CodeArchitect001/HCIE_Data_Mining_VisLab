## 三、模块详细梳理

### 第一章 数据挖掘介绍

#### 1.1 数据挖掘概述

##### 1.1.1 数据挖掘概念

**定义**：数据挖掘（Data Mining）是从大量、不完全、有噪声、模糊的原始数据中，提取隐含在其中的、人们事先不知道的、但又是潜在有用的信息和知识的过程。

**与相关概念的关系**：

| 概念 | 定义 | 与数据挖掘的关系 |
| --- | --- | --- |
| 大数据（Big Data） | 具有Volume（大量）、Velocity（高速）、Variety（多样）、Value（价值）特征的巨量数据集合 | 大数据是数据挖掘的**数据基础**和**处理对象** |
| 数据分析（Data Analysis） | 用统计分析方法对数据进行描述、探索和验证 | 数据分析是数据挖掘的**前置步骤**和**子集** |
| 机器学习（Machine Learning） | 让计算机通过经验（数据）自动改进性能的算法研究 | 机器学习是数据挖掘的**核心技术手段** |
| 人工智能（AI） | 使机器模拟人类智能的科学与技术 | AI是数据挖掘的**上层领域**，数据挖掘是AI在数据分析领域的具体应用 |

**覆盖学科**：
- **统计学**：假设检验、概率分布、回归分析
- **机器学习**：算法模型、模式识别
- **数据库技术**：数据存储、查询优化、数据仓库
- **可视化技术**：数据展示、交互探索

##### 1.1.2 应用场景（MECE分类）

| 行业领域 | 典型应用场景 | 常用算法 |
| --- | --- | --- |
| **金融** | 信用评分、欺诈检测、风险预测、量化投资 | 逻辑回归、随机森林、Xgboost |
| **警务** | 犯罪预测、嫌疑人画像、案件关联分析 | 聚类、关联规则、神经网络 |
| **数字政府** | 政务数据治理、政策效果评估、公共资源优化 | 回归、时间序列、决策树 |
| **智慧园区** | 能耗预测、人流分析、安防预警 | 聚类、分类、异常检测 |
| **电商** | 用户画像、商品推荐、销量预测、点击率预估 | 协同过滤、逻辑回归、GBDT+LR |

#### 1.4 CRISP-DM流程

**CRISP-DM**（Cross-Industry Standard Process for Data Mining）是跨行业数据挖掘标准流程，是业界公认的数据挖掘项目方法论。

##### 1.4.1 六阶段流程图

```

│    │Business  │    │  Data    │    │  Data    │            │
│    │Understanding│  │Understanding│  │Preparation│           │
│    └──────────┘    └──────────┘    └────┬─────┘            │
│                                         │                  │
│    ┌──────────┐    ┌──────────┐         │                  │
│    │  模型实施  │←───│   评估   │←────────┘                  │
│    │Deployment│    │Evaluation│                             │
│    └──────────┘    └────┬─────┘                             │
│                         │                                   │
│                         ↓ (反馈迭代)                         │
│                    ┌──────────┐                              │
│                    │   建模   │                              │
│                    │ Modeling │                              │
│                    └──────────┘                              │
└─────────────────────────────────────────────────────────────┘
```

##### 1.4.2 各阶段详解

| 阶段 | 核心目标 | 关键活动 | 交付物 |
| --- | --- | --- | --- |
| **商业理解（Business Understanding）** | 明确业务目标，转化为数据挖掘目标 | 需求调研、目标定义、 success criteria制定 | 项目计划书、业务目标文档 |
| **数据理解（Data Understanding）** | 熟悉数据特征，发现数据质量问题 | 数据收集、数据描述、数据探索、质量评估 | 数据质量报告、数据描述文档 |
| **数据准备（Data Preparation）** | 构建最终建模数据集 | 数据清洗、特征工程、数据集划分（训练/验证/测试） | 建模数据集、特征说明文档 |

**重要特征**：CRISP-DM是一个**循环迭代**过程，评估结果可能反馈到任何前面阶段进行优化。

### 第二章 数据预处理与特征工程

#### 2.1 数据预处理概述

##### 2.1.1 数据对象和属性类型（MECE分类）

| 属性类型 | 定义 | 特征 | 示例 |
| --- | --- | --- | --- |
| **标称属性（Nominal）** | 仅作为名称或标签，无顺序含义 | =, ≠ 运算；不可比较大小 | 颜色{红,绿,蓝}、邮政编码 |
| **二元属性（Binary）** | 只有两个状态的标称属性 | 对称/非对称两种子类型 | 性别{男,女}、是否吸烟{是,否} |
| **序数属性（Ordinal）** | 具有有意义的顺序或排名 | <, > 运算；差值无意义 | 成绩等级{A,B,C}、满意度{高,中,低} |
| **数值属性（Numeric）** | 可度量的定量值 | +, -, *, / 运算；区间/比率两种 | 温度、身高、收入、年龄 |

**属性类型之间的MECE关系**：
- 标称、二元、序数、数值四类互斥，每个属性必属于且仅属于一类
- 二元是标称的特殊情况，但因特殊重要性单独列出
- 数值属性进一步分为区间标度和比率标度

##### 2.1.2 数据集类型

| 数据集类型 | 描述 | 应用场景 |
| --- | --- | --- |
| 记录数据 | 由多条等长记录组成的数据集 | 最常见，如CSV表格 |
| 图数据 | 节点和边的集合，表达实体关系 | 社交网络、知识图谱 |
| 有序数据 | 数据项之间存在时间或空间顺序 | 时间序列、DNA序列 |
| 空间数据 | 包含地理空间信息的数据 | 地理位置分析、GIS |

##### 2.1.3 汇总统计

| 统计量 | 定义 | 用途 |
| --- | --- | --- |
| 均值（Mean） | 所有数值的平均 | 数据中心趋势度量 |
| 中位数（Median） | 排序后中间位置的值 | 抗极端值影响的中心度量 |
| 众数（Mode） | 出现频率最高的值 | 类别数据的最频繁项 |
| 方差（Variance） | 各数据与均值差的平方的平均 | 数据离散程度度量 |
| 标准差（Std） | 方差的平方根 | 与原始数据同量纲的离散度量 |
| 四分位数（Quartile） | 将数据分为4等份的3个分割点 | 数据分布位置度量 |
| 极差（Range） | 最大值-最小值 | 数据范围度量 |

#### 2.2 数据清洗

##### 2.2.1 缺失值处理（MECE策略矩阵）

| 处理策略 | 方法说明 | 适用场景 | 优缺点 |
| --- | --- | --- | --- |
| **删除法** | 删除包含缺失值的行或列 | 缺失比例低（<5%）且随机缺失 | 简单直接；可能损失信息 |
| **均值填充（Mean）** | 用该特征的均值填充 | 数值型特征，近似正态分布 | 保持均值不变；降低方差 |
| **中位数填充（Median）** | 用该特征的中位数填充 | 数值型特征，存在偏态或离群点 | 抗离群点影响 |
| **众数填充（Mode）** | 用该特征的众数填充 | 类别型特征 | 保持最频繁类别 |
| **KNN填充** | 用K个最近邻的加权均值填充 | 特征间存在相关性 | 利用特征关系；计算量大 |
| **回归填充** | 用其他特征建立回归模型预测缺失值 | 缺失特征与其他特征强相关 | 精度较高；可能引入估计误差 |
| **变量映射** | 将缺失作为一种特殊类别处理 | 类别型特征，缺失可能有业务含义 | 保留缺失信息；增加维度 |
| **不处理** | 某些算法（如Xgboost）可自动处理缺失 | 使用树模型等内置缺失处理能力的算法 | 依赖算法能力 |

**缺失值处理决策树**：

```
缺失值处理决策
├── 缺失比例 > 50%？
│   ├── 是 → 考虑删除该特征
│   └── 否 → 继续判断
├── 特征类型？
│   ├── 类别型 → 众数填充 / 变量映射（新增类别）
│   └── 数值型 → 继续判断
├── 分布类型？
│   ├── 正态分布 → 均值填充
│   ├── 偏态分布 → 中位数填充
│   └── 存在特征相关性 → KNN填充 / 回归填充
└── 使用模型是否内置缺失处理？
    ├── 是（如Xgboost）→ 可不处理
    └── 否 → 选择上述方法之一
```

##### 2.2.2 离群点处理（MECE检测方法）

| 检测方法 | 原理 | 适用场景 | 操作方式 |
| --- | --- | --- | --- |
| **散点图法** | 可视化观察，人工识别偏离主体的点 | 低维数据（2D/3D），探索性分析 | 绘制散点图，目视识别 |
| **3σ原则** | 数据超出(μ-3σ, μ+3σ)范围的视为离群点 | 正态分布或近似正态分布的数据 | 计算均值μ和标准差σ，筛选超出范围的值 |
| **箱型图法（IQR）** | 超出[Q1-1.5*IQR, Q3+1.5*IQR]范围的为离群点 | 任意连续分布，不受极端值影响 | 计算四分位数和IQR，确定上下界 |
| **分类模型检测** | 用决策树等模型识别异常模式 | 高维数据，复杂分布 | 训练分类器或使用异常检测算法 |

**箱型图离群点判定公式**：
- 下界 = Q1 - 1.5 × IQR
- 上界 = Q3 + 1.5 × IQR
- 其中 IQR = Q3 - Q1（四分位距）
- 超出上下界的点为离群点

**离群点处理策略**：
- 删除：离群点为数据采集错误
- 替换：用上下界值或均值/中位数替换
- 保留：离群点包含重要信息（如欺诈检测）
- 转换：通过对数等变换降低离群点影响

##### 2.2.3 遗漏值处理

遗漏值与缺失值的区别：
- **缺失值**：数据中存在空值/NaN，明确记录为缺失
- **遗漏值**：数据看似有值，但实际未记录或记录不全（如默认值0代替未输入）

处理方法：
- 业务规则识别：结合业务知识识别不合理的"伪值"
- 一致性检查：通过交叉验证发现逻辑矛盾
- 模式识别：识别系统默认填充值的规律

#### 2.3 数据转换

##### 2.3.1 特征缩放（MECE方法）

| 方法 | 公式 | 输出范围 | 适用场景 | 对离群点敏感度 |
| --- | --- | --- | --- | --- |
| **标准化（Standardization/Z-score）** | z = (x - μ) / σ | 均值0，标准差1；无固定范围 | 特征近似正态分布；需要距离计算的算法（KNN、SVM） | 敏感 |
| **归一化（Normalization/Min-Max）** | x' = (x - min) / (max - min) | [0, 1] | 需要固定范围输出；神经网络输入 | 非常敏感 |
| **均值归一化** | x' = (x - μ) / (max - min) | [-1, 1] | 需要以0为中心的对称范围 | 敏感 |
| **正则化（Normalization/单位向量）** | x' = x / ||x|| | 单位长度 | 需要比较向量方向而非 magnitude | 敏感 |

**缩放方法选择决策**：

```
特征缩放选择
├── 算法是否对特征尺度敏感？
│   ├── 否（如决策树、朴素贝叶斯）→ 不需要缩放
│   └── 是（如KNN、SVM、神经网络、线性回归）→ 继续判断
├── 特征分布？
│   ├── 近似正态 → 标准化（Z-score）
│   └── 未知/非正态 → 继续判断
├── 是否存在离群点？
│   ├── 是 → 稳健标准化（用中位数和IQR）
│   └── 否 → 继续判断
└── 是否需要固定范围？
    ├── 是 → Min-Max归一化
    └── 否 → 标准化
```

##### 2.3.2 连续型离散化（MECE方法）

| 方法类别 | 具体方法 | 原理 | 特点 |
| --- | --- | --- | --- |
| **二值化（Binarization）** | 设定阈值，高于为1，低于为0 | 将连续值转为0/1 | 简单，适合特定业务场景 |
| **无监督分箱** | **等距分箱（Equal-width）** | 将值域均分为K个区间 | 每区间宽度相同；受离群点影响 |
|  | **等频分箱（Equal-frequency/分位数）** | 每区间包含近似相同样本数 | 每区间样本均衡；区间宽度不等 |
|  | **聚类分箱（K-Means）** | 用K-Means聚类中心划分区间 | 基于数据分布自动划分 |
| **有监督分箱** | **卡方分箱（Chi-Merge/Chi-square）** | 基于卡方检验合并具有相似类别分布的相邻区间 | 考虑目标变量，保留预测信息 |

**离散化方法对比**：

| 维度 | 等距分箱 | 等频分箱 | 聚类分箱 | 卡方分箱 |
| --- | --- | --- | --- | --- |
| 是否用目标标签 | 否 | 否 | 否 | 是 |
| 分箱依据 | 值域范围 | 样本频率 | 数据密度 | 类别一致性 |
| 对离群点鲁棒性 | 差 | 好 | 好 | 好 |
| 计算复杂度 | 低 | 中 | 高 | 高 |
| 适用场景 | 均匀分布 | 偏态分布 | 多峰分布 | 分类任务特征工程 |

##### 2.3.3 离散型编码（MECE方法）

| 编码方法 | 原理 | 输出特征数 | 适用场景 | 优缺点 |
| --- | --- | --- | --- | --- |
| **独热编码（One-Hot Encoding）** | 每个类别创建一个二元特征，属于该类为1，否则为0 | 原类别数 | 类别间无顺序关系（标称属性） | 无顺序假设；会增加特征维度 |
| **标签编码（Label Encoding）** | 将类别映射为整数值（0, 1, 2, …） | 1 | 类别间有顺序关系（序数属性）；树模型 | 引入数值顺序假设；可能误导线性模型 |
| **哑编码（Dummy Encoding）** | 类似One-Hot，但省略一个参考类别避免多重共线性 | 原类别数-1 | 回归分析、需要避免共线性 | 减少一维；保留信息完整 |

**编码方法选择决策**：

```
离散型编码选择
├── 属性类型？
│   ├── 序数属性（有顺序） → Label Encoding
│   └── 标称属性（无顺序） → 继续判断
├── 模型类型？
│   ├── 树模型（决策树、随机森林、Xgboost） → Label Encoding也可
│   └── 线性模型、神经网络、距离模型 → One-Hot / Dummy Encoding
└── 类别数量？
    ├── 类别数少（<10） → One-Hot Encoding
    └── 类别数多（>=10） → 考虑目标编码、嵌入编码或分箱
```

##### 2.3.5 偏态处理（MECE方法）

| 方法 | 公式 | 适用偏态类型 | 限制条件 |
| --- | --- | --- | --- |
| **平方根法（Square Root）** | sqrt(x) 或 sqrt(x+c) | 右偏（正偏态），中度偏态 | x >= 0 |
| **对数转换（Log Transformation）** | log(x) 或 log(x+1) | 右偏，重度偏态 | x > 0；log(x+1)可处理x=0 |

**偏态处理方法选择**：

```
│   └── 否 → 继续判断
├── 偏态程度？
│   ├── 中度偏态 → 平方根变换
│   └── 重度偏态 → 对数变换 / Box-Cox
└── 是否需要自动优化？
    ├── 是 → Box-Cox（自动搜索lambda）

```

#### 2.5 特征工程

##### 2.5.1 特征选择定义

**特征选择（Feature Selection）**：从原始特征集合中选择一个最优子集，以达到降维、去噪、提升模型性能、增强可解释性的目的。

**特征选择 vs 降维**：

| 维度 | 特征选择（Feature Selection） | 降维（Dimensionality Reduction） |
| --- | --- | --- |
| **核心思想** | 选择原始特征的子集 | 将原始特征映射到低维空间 |
| **输出特征** | 原始特征的子集，保留原始含义 | 新的综合特征，可能无直接业务含义 |
| **代表方法** | Filter/Wrapper/Embedded | PCA、LDA、t-SNE |
| **特征保留** | 保留原始特征名称和含义 | 不保留原始特征含义 |
| **计算复杂度** | 一般较低 | 一般较高 |
| **可解释性** | 高（保留原始特征含义） | 低（新特征难以解释） |

##### 2.5.2 特征选择方法（MECE三类）

```
特征选择方法体系
|-- Filter方法（过滤法）【基于统计度量，独立于模型】
|   |-- 方差选择法
|   |-- 相关系数法
|   |-- 卡方检验
|   |-- 互信息法
|
|-- Wrapper方法（包装法）【基于模型性能迭代选择】
|   |-- RFE（递归特征消除）
|   |-- 前向选择
|   |-- 后向消除
|
|-- Embedded方法（嵌入法）【在模型训练中自动选择】
    |-- L1正则化（Lasso）
    |-- 基于树模型的特征重要性
    |-- 随机森林特征重要性
```

**Filter方法详解**：

| 方法 | 原理 | 适用场景 | 特点 |
| --- | --- | --- | --- |
| **方差选择法** | 删除方差低于阈值的特征 | 删除几乎不变的特征 | 简单快速；仅考虑特征本身，不考虑目标变量 |
| **相关系数法** | 计算特征与目标变量的相关系数，选择绝对值高的 | 线性关系的特征筛选 | 仅捕捉线性关系；对Pearson相关系数要求正态分布 |
| **卡方检验（Chi-square）** | 检验类别特征与类别目标变量的独立性 | 类别型特征的分类任务 | 仅适用于类别型特征；选择卡方统计量大的特征 |
| **互信息法（Mutual Information）** | 度量特征与目标变量之间的互信息 | 捕捉非线性关系 | 能捕捉非线性依赖；计算复杂度较高 |

**Wrapper方法详解**：

| 方法 | 原理 | 过程 | 特点 |
| --- | --- | --- | --- |
| **RFE（递归特征消除）** | 从全部特征开始，递归地移除最弱特征 | 1. 训练模型得到特征重要性；2. 移除最弱特征；3. 重复直到达到指定特征数 | 考虑特征组合；计算量大；与具体模型绑定 |

**Embedded方法详解**：

| 方法 | 原理 | 代表模型 | 特点 |
| --- | --- | --- | --- |
| **L1正则化（Lasso）** | L1惩罚使部分特征系数压缩为0，实现自动选择 | Lasso回归、Logistic回归+L1 | 同时进行特征选择和模型训练；适合高维稀疏数据 |
| **基于树模型的特征重要性** | 通过特征在树分裂中的贡献度（信息增益/GINI减少）评估重要性 | 决策树、随机森林、Xgboost | 能捕捉非线性关系和特征交互；随机森林通过多棵树平均降低方差 |

**三种特征选择方法对比**：

| 维度 | Filter | Wrapper | Embedded |
| --- | --- | --- | --- |
| 是否依赖模型 | 否 | 是 | 是 |
| 计算速度 | 快 | 慢（需多次训练模型） | 中（单次训练） |
| 过拟合风险 | 低 | 高 | 中 |
| 特征交互捕捉 | 无 | 有 | 部分有（树模型） |
| 泛化能力 | 好 | 一般 | 好 |
| 最佳实践 | 快速初筛 | 精细优化 | 推荐首选 |

### 第三章 回归与分类问题建模

#### 3.1 基础概念框架

##### 3.1.1 学习范式（MECE分类）

| 学习范式 | 数据标签 | 核心机制 | 代表算法 | 应用场景 |
| --- | --- | --- | --- | --- |
| **监督学习** | 全部有标签 | 学习输入到输出的映射函数 | 逻辑回归、决策树、SVM、线性回归 | 分类、回归预测 |
| **无监督学习** | 无标签 | 发现数据的内在结构和模式 | K-Means、层次聚类、Apriori | 聚类、降维、关联分析 |
| **半监督学习** | 少量标签+大量无标签 | 利用未标注数据辅助学习 | 自训练、协同训练、标签传播 | 标注成本高的场景 |
| **强化学习** | 环境反馈（奖励/惩罚） | 学习最优动作策略 | Q-Learning、DQN、Policy Gradient | 游戏、机器人、推荐系统 |

##### 3.1.2 偏差（Bias）与方差（Variance）

**核心概念**：

| 概念 | 定义 | 低/高表现 | 原因 |
| --- | --- | --- | --- |
| **偏差（Bias）** | 模型预测期望值与真实值之间的差异 | 高偏差->欠拟合；低偏差->拟合充分 | 模型过于简单，假设太强 |
| **方差（Variance）** | 模型在不同训练集上预测结果的波动程度 | 高方差->过拟合；低方差->稳定性好 | 模型过于复杂，对训练数据敏感 |

**偏差-方差分解**：

```
泛化误差 = 偏差^2 + 方差 + 不可约误差（噪声）
        = Bias^2 + Variance + Irreducible Error
```

**偏差-方差权衡（Trade-off）**：
- 模型复杂度↑ -> 偏差↓，方差↑
- 模型复杂度↓ -> 偏差↑，方差↓
- **目标**：找到偏差和方差都较小的平衡点

##### 3.1.3 过拟合与欠拟合

| 概念 | 定义 | 表现 | 解决方案 |
| --- | --- | --- | --- |
| **欠拟合（Underfitting）** | 模型过于简单，未能捕捉数据规律 | 训练集和测试集表现都差 | 增加模型复杂度、增加特征、减少正则化 |
| **过拟合（Overfitting）** | 模型过于复杂，过度学习训练集噪声 | 训练集表现好，测试集表现差 | 增加数据量、正则化、特征选择、交叉验证、简化模型 |

##### 3.1.4 泛化能力

**定义**：模型对未见过的新数据的预测能力。

**提升泛化能力的方法（MECE）**：
1. **数据层面**：增加训练数据量、数据增强
2. **特征层面**：特征选择、降维、特征工程
3. **模型层面**：正则化、简化模型结构
4. **训练层面**：交叉验证、早停（Early Stopping）
5. **集成层面**：Bagging、Boosting、Stacking

##### 3.1.5 凸函数

**定义**：函数f(x)为凸函数，当且仅当对于任意x1, x2和lambda属于[0,1]，满足：

```
f(lambda*x1 + (1-lambda)*x2) <= lambda*f(x1) + (1-lambda)*f(x2)
```

**在机器学习中的意义**：
- 凸函数的局部最小值即为全局最小值
- 线性回归、逻辑回归（在对数似然下）、SVM的损失函数都是凸函数
- 神经网络等非凸优化需要更复杂的优化策略

#### 3.2 分类算法

##### 3.2.1 逻辑回归（Logistic Regression）

**核心原理**：通过Sigmoid函数将线性回归的输出映射到(0,1)区间，表示概率。

**Logistic分布**：

```
F(x) = P(X <= x) = 1 / (1 + e^(-(x-mu)/gamma))
```

**Sigmoid函数**：

```
sigma(z) = 1 / (1 + e^(-z))
```

- 输出范围：(0, 1)
- 性质：sigma(-z) = 1 - sigma(z)
- 当z=0时，sigma(z)=0.5

**模型形式**：

```
P(y=1|x) = sigma(w^T x + b) = 1 / (1 + e^(-(w^T x + b)))
```

**损失函数（对数损失/交叉熵损失）**：

```
L(w) = -1/N * sum[yi * log(yi_hat) + (1-yi) * log(1-yi_hat)]
```

**参数估计方法：极大似然估计（MLE）**：
1. 构建似然函数：L(w) = prod P(yi|xi;w)
2. 取对数得对数似然：log L(w) = sum [yi log(yi_hat) + (1-yi)log(1-yi_hat)]
3. 最大化对数似然等价于最小化交叉熵损失
4. 使用梯度下降法求解最优参数w

**关键特点**：

| 特点 | 说明 |
| --- | --- |
| 模型类型 | 线性分类器（虽然是"回归"，但用于分类） |
| 输出解释 | 概率输出，可解释性强 |
| 多分类扩展 | One-vs-Rest（OvR）或 Softmax回归 |
| 优点 | 简单高效、可解释性强、不易过拟合、输出概率 |
| 缺点 | 只能拟合线性决策边界、对特征工程依赖较大 |
| 适用场景 | 信用评分、点击率预估、疾病诊断、二分类问题 |

##### 3.2.2 KNN（K-Nearest Neighbors，K近邻算法）

**核心原理**："近朱者赤，近墨者黑"——样本的类别由其K个最近邻的多数类别决定。

**三要素（MECE）**：

| 要素 | 说明 | 选择 |
| --- | --- | --- |
| **K值选择** | 决定参考邻居的数量 | K小->模型复杂->易过拟合；K大->模型简单->易欠拟合；通常通过交叉验证选择 |
| **距离度量** | 计算样本间距离的方法 | 欧氏距离（连续特征）、曼哈顿距离（高维/离散特征）、闵可夫斯基距离 |
| **分类决策规则** | 如何由K个邻居决定类别 | 多数表决（分类）、平均值（回归）、加权投票（距离越近权重越大） |

**距离度量公式**：

| 距离类型 | 公式 | 特点 |
| --- | --- | --- |
| **欧氏距离（Euclidean）** | d = sqrt(sum(xi - yi)^2) | 最常用；直观的几何距离 |
| **曼哈顿距离（Manhattan）** | d = sum \|xi - yi\| | 高维数据更稳健 |
| **闵可夫斯基距离** | d = (sum \|xi - yi\|^p)^(1/p) | 通用形式；p=1为曼哈顿，p=2为欧氏 |

**算法流程**：
1. 计算待分类样本与训练集中所有样本的距离
2. 按距离排序，选取K个最近邻
3. 根据K个邻居的类别进行投票
4. 得票最多的类别作为预测结果

**关键特点**：

| 特点 | 说明 |
| --- | --- |
| 模型类型 | 非参数方法、惰性学习（Lazy Learning） |
| 训练阶段 | 无显式训练，仅存储训练数据 |
| 预测阶段 | 需要计算与所有训练样本的距离，计算量大 |
| 优点 | 简单直观、无需训练、适应复杂决策边界 |
| 缺点 | 预测慢（O(n)）、对K值敏感、需要特征缩放、内存消耗大 |
| 适用场景 | 小规模数据集、推荐系统、图像识别 |

##### 3.2.3 朴素贝叶斯（Naive Bayes）

**核心原理**：基于贝叶斯定理和类条件独立假设进行分类。

**贝叶斯定理**：

```
P(Y|X) = P(X|Y) * P(Y) / P(X)
```

- P(Y|X)：后验概率（给定特征X，类别Y的概率）
- P(X|Y)：似然（给定类别Y，特征X的概率）
- P(Y)：先验概率（类别Y的初始概率）
- P(X)：证据因子（特征X出现的概率）

**类条件独立假设（"朴素"的来源）**：

```
P(X|Y) = P(x1|Y) * P(x2|Y) * ... * P(xn|Y) = prod P(xi|Y)
```

即假设各特征在给定类别条件下相互独立。

**工作过程**：
1. 计算每个类别的先验概率 P(Y)
2. 计算每个特征在每个类别下的条件概率 P(xi|Y)
3. 对于新样本，计算每个类别的后验概率 P(Y|X)
4. 选择后验概率最大的类别作为预测结果

**常见变体**：

| 变体 | 特征假设 | 适用场景 |
| --- | --- | --- |
| **高斯朴素贝叶斯** | 连续特征服从高斯分布 | 连续型特征的分类 |
| **多项式朴素贝叶斯** | 特征为离散计数 | 文本分类（词频） |
| **伯努利朴素贝叶斯** | 特征为二元值（0/1） | 文本分类（词是否出现） |

**关键特点**：

| 特点 | 说明 |
| --- | --- |
| 模型类型 | 概率生成模型 |
| 优点 | 训练和预测速度快、对缺失数据不敏感、适合高维数据、对小样本效果好 |
| 缺点 | 类条件独立假设通常不成立、特征相关时效果下降 |
| 适用场景 | 垃圾邮件过滤、文本分类、情感分析、疾病诊断 |

##### 3.2.4 决策树（Decision Tree）

**核心原理**：通过递归地选择最优特征进行分裂，构建树形结构的分类模型。

**关键概念**：

| 概念 | 定义 | 公式 |
| --- | --- | --- |
| **信息熵（Entropy）** | 度量样本集合的不纯度 | H(S) = -sum pi * log2(pi) |
| **信息增益（Information Gain）** | 特征分裂后熵的减少量 | IG(S,A) = H(S) - sum(\|Sv\|/\|S\|)H(Sv) |
| **信息增益率（Gain Ratio）** | 信息增益除以分裂信息，解决信息增益偏向多值特征的问题 | GR(S,A) = IG(S,A) / SplitInfo(A) |
| **GINI指数** | 度量数据集的不纯度，从集合中随机抽取两个样本类别不一致的概率 | GINI(S) = 1 - sum pi^2 |

**三种经典算法对比（MECE）**：

| 算法 | 分裂准则 | 树类型 | 特征类型 | 特点 |
| --- | --- | --- | --- | --- |
| **ID3** | 信息增益 | 多叉树 | 只能处理类别特征 | 最早；偏向多值特征；无剪枝 |
| **C4.5** | 信息增益率 | 多叉树 | 类别+连续特征 | ID3改进；可处理连续值；有剪枝 |
| **CART** | GINI指数 | 二叉树 | 类别+连续特征 | 既可分类也可回归；最常用；生成二叉树 |

**剪枝策略（MECE）**：

| 剪枝类型 | 时机 | 方法 | 特点 |
| --- | --- | --- | --- |
| **预剪枝（Pre-pruning）** | 树生成过程中 | 设定阈值（最大深度、最小样本数、最小信息增益），提前停止分裂 | 速度快；可能欠拟合（过早停止） |
| **后剪枝（Post-pruning）** | 树完全生成后 | 自底向上，用验证集评估是否合并子树 | 效果更好；计算量大；通常更优 |

**决策树生成流程**：
1. 从根节点开始，包含全部训练样本
2. 对每个特征，计算分裂准则（信息增益/增益率/GINI）
3. 选择最优分裂特征和分裂点
4. 将数据分到不同子节点
5. 递归地对每个子节点重复步骤2-4
6. 满足停止条件时停止（预剪枝）或生成后剪枝
7. 将叶节点标记为多数类别（分类）或平均值（回归）

**关键特点**：

| 特点 | 说明 |
| --- | --- |
| 优点 | 可解释性强（白盒模型）、能处理非线性关系、可处理类别和数值特征、无需特征缩放 |
| 缺点 | 容易过拟合、不稳定（数据微小变化导致不同树）、偏向多值特征（ID3）、可能产生偏斜树 |
| 解决方案 | 剪枝、随机森林（集成多棵树） |

##### 3.2.5 SVM（支持向量机）

**核心原理**：寻找一个最优超平面，使得不同类别的样本之间的间隔（Margin）最大化。

**核心概念**：

| 概念 | 定义 |
| --- | --- |
| **超平面（Hyperplane）** | 将特征空间划分为两部分的决策边界：w^T x + b = 0 |
| **间隔（Margin）** | 超平面到最近样本点（支持向量）的距离 |
| **支持向量（Support Vector）** | 位于间隔边界上的样本点，决定超平面位置 |
| **硬间隔（Hard Margin）** | 要求所有样本被正确分类，无容错 |
| **软间隔（Soft Margin）** | 允许部分样本分类错误，引入松弛变量和惩罚参数C |

**线性分类SVM**：

目标：最大化 Margin = 2 / ||w||
等价于：最小化 1/2 * ||w||^2

约束条件：yi(w^T xi + b) >= 1 （硬间隔）
yi(w^T xi + b) >= 1 - xi_i，xi_i >= 0 （软间隔）

**非线性分类SVM**：

通过核函数将低维数据映射到高维特征空间，在高维空间中线性可分。

**核函数（MECE分类）**：

| 核函数 | 公式 | 适用场景 | 参数 |
| --- | --- | --- | --- |
| **线性核（Linear）** | K(x,y) = x^T y | 特征数多、数据量大、线性可分 | 无 |
| **多项式核（Polynomial）** | K(x,y) = (gamma*x^T y + r)^d | 已知特征间存在多项式关系 | 次数d、gamma、r |
| **RBF核/高斯核（Radial Basis Function）** | K(x,y) = exp(-gamma \|\|x-y\|\|^2) | 通用首选，非线性问题 | gamma |
| **Sigmoid核** | K(x,y) = tanh(gamma*x^T y + r) | 类似神经网络激活函数 | gamma、r |

**核函数选择建议**：

```
核函数选择决策
├── 特征维度高（>1000）且样本少？
│   ├── 是 → 线性核（避免过拟合）
│   └── 否 → 继续判断
├── 样本量中等、特征数中等？
│   └── 是 → RBF核（通用首选）
└── 是否已知特征间存在特定关系？
    ├── 多项式关系 → 多项式核
    └── 神经网络-like → Sigmoid核
```

**优化方法**：

| 方法 | 说明 |
| --- | --- |
| **拉格朗日优化** | 将有约束优化问题转化为无约束的对偶问题 |
| **SMO（Sequential Minimal Optimization）** | 每次选择两个拉格朗日乘子进行优化，将大问题分解为一系列最小子问题；SVM的标准训练算法 |

**关键特点**：

| 特点 | 说明 |
| --- | --- |
| 优点 | 泛化能力强（结构风险最小化）、适合高维数据、核函数灵活、最终模型仅依赖支持向量 |
| 缺点 | 大数据集训练慢（O(n^2)~O(n^3)）、对核函数和参数敏感、多分类需要额外策略、概率输出需要额外计算 |
| 适用场景 | 文本分类、图像识别、生物信息学、高维数据分类 |
| 多分类策略 | One-vs-One（两两比较）、One-vs-Rest（一对多） |

#### 3.3 回归算法

##### 3.3.1 线性回归（Linear Regression）

**核心原理**：假设目标变量y与特征x之间存在线性关系，拟合最优线性模型。

**模型形式**：

```
y = w^T x + b = w1*x1 + w2*x2 + ... + wn*xn + b
```

**参数估计方法**：

| 方法 | 原理 | 公式 | 特点 |
| --- | --- | --- | --- |
| **最小二乘法（OLS）** | 最小化预测值与真实值之差的平方和 | L = sum(yi - yi_hat)^2 = sum(yi - w^T xi)^2 | 解析求解；对离群点敏感 |
| **梯度下降法** | 沿损失函数负梯度方向迭代更新参数 | w := w - alpha * dL/dw | 通用性强；需选择学习率alpha |
| **正规方程** | 直接解析求解最优参数 | w = (X^T X)^(-1) X^T y | 无需迭代；X^T X需可逆；计算复杂度高O(n^3) |

**梯度下降变体（MECE）**：

| 变体 | 更新方式 | 每次迭代样本数 | 优点 | 缺点 |
| --- | --- | --- | --- | --- |
| **批量梯度下降（BGD）** | 使用全部样本计算梯度 | N（全部） | 稳定收敛，准确梯度 | 计算量大，内存占用大 |
| **随机梯度下降（SGD）** | 随机选取一个样本计算梯度 | 1 | 计算快，可逃离局部最优 | 波动大，收敛不稳定 |
| **小批量梯度下降（Mini-batch）** | 选取一小批样本计算梯度 | b（如32, 64, 128） | 平衡稳定性和效率；最常用 | 需选择batch size |

**关键特点**：

| 特点 | 说明 |
| --- | --- |
| 优点 | 简单直观、计算高效、可解释性强（系数代表特征重要性）、是复杂模型的基础 |
| 缺点 | 只能拟合线性关系、对多重共线性敏感、对离群点敏感 |
| 假设条件 | 线性关系、误差正态分布、特征独立、同方差性 |

##### 3.3.2 多项式回归（Polynomial Regression）

**核心原理**：通过引入特征的幂次和交叉项，将线性回归扩展到非线性关系拟合。

**模型形式**：

```
y = w0 + w1*x + w2*x^2 + w3*x^3 + ... + wn*x^n + epsilon
```

**关键特点**：

| 特点 | 说明 |
| --- | --- |
| 本质 | 对特征进行多项式变换后，再用线性回归拟合 |
| 多项式次数 | 次数越高，模型越复杂，越容易过拟合 |
| 优点 | 能拟合非线性关系、实现简单 |
| 缺点 | 高次多项式易过拟合、特征维度急剧增加 |
| 注意事项 | 必须配合正则化或交叉验证选择最优次数 |

##### 3.3.3 Ridge回归（L2正则化）

**核心原理**：在普通最小二乘的基础上增加L2正则化项，防止过拟合。

**损失函数**：

```
L = sum(yi - yi_hat)^2 + alpha * sum(wj^2)
    ↑                         ↑
  均方误差（MSE）            L2正则化项（Ridge惩罚）
```

**关键特点**：

| 特点 | 说明 |
| --- | --- |
| 别名 | Tikhonov正则化、L2正则化回归 |
| 正则化效果 | 收缩系数使其接近0但不等于0 |
| 超参数alpha | alpha=0等价于普通线性回归；alpha越大，正则化越强，系数越接近0 |
| 优点 | 解决多重共线性、防止过拟合、有解析解 |
| 缺点 | 不能进行特征选择（系数不会精确为0） |

##### 3.3.4 LASSO回归（L1正则化）

**核心原理**：在线性回归损失函数中增加L1正则化项，实现特征选择。

**损失函数**：

```
L = sum(yi - yi_hat)^2 + alpha * sum(\|wj\|)
    ↑                         ↑
  均方误差（MSE）            L1正则化项（LASSO惩罚）
```

**关键特点**：

| 特点 | 说明 |
| --- | --- |
| 全称 | Least Absolute Shrinkage and Selection Operator |
| 正则化效果 | 使部分不重要的特征系数精确为0，实现特征选择 |
| 超参数alpha | alpha越大，越多特征系数变为0 |
| 优点 | 同时进行回归和特征选择、产生稀疏模型、适合高维数据 |
| 缺点 | 存在多重共线性时不稳定、非凸优化（绝对值函数） |
| 与Ridge对比 | Ridge使系数收缩；LASSO使部分系数为0 |

##### 3.3.5 ElasticNet回归（L1+L2组合正则化）

**核心原理**：结合L1和L2正则化的优点，同时实现特征选择和系数收缩。

**损失函数**：

```
L = sum(yi - yi_hat)^2 + alpha * [rho * sum(\|wj\|) + (1-rho) * sum(wj^2) / 2]
    ↑                         ↑
    L1部分                   L2部分
```

**关键特点**：

| 特点 | 说明 |
| --- | --- |
| 超参数alpha | 整体正则化强度 |
| 超参数rho（l1_ratio） | L1和L2的比例；rho=1为纯LASSO，rho=0为纯Ridge |
| 优点 | 兼具LASSO特征选择和Ridge稳定性；处理相关特征组比LASSO更好 |
| 适用场景 | 特征数远大于样本数、特征间存在相关性 |

**三种回归正则化对比**：

| 维度 | 线性回归 | Ridge回归 | LASSO回归 | ElasticNet |
| --- | --- | --- | --- | --- |
| 正则化项 | 无 | L2 | L1 | L1 + L2 |
| 系数结果 | 任意值 | 接近0但不为0 | 部分精确为0 | 部分精确为0 |
| 特征选择 | 无 | 无 | 有 | 有 |
| 多重共线性 | 敏感 | 稳健 | 不稳定 | 稳健 |
| 适用场景 | 简单问题 | 共线性问题 | 高维特征选择 | 高维+相关特征 |

##### 3.3.6 SVR（支持向量回归）

**核心原理**：SVM的回归版本，寻找一个函数，使得大部分训练样本的预测误差在epsilon范围内，同时模型尽可能平坦。

**核心概念**：

| 概念 | 说明 |
| --- | --- |
| **epsilon-不敏感带** | 预测值与真实值之差在+-epsilon范围内不计损失 |
| **间隔（Margin）** | 类似分类SVM，最大化模型平坦度 |
| **支持向量** | 位于epsilon-带外或边界上的样本 |
| **惩罚参数C** | 控制对超出epsilon范围的样本的惩罚程度 |

**关键特点**：

| 特点 | 说明 |
| --- | --- |
| 优点 | 适合非线性回归（核函数）、对离群点鲁棒（epsilon-带）、泛化能力强 |
| 缺点 | 大数据集训练慢、参数（C, epsilon, 核函数）需调优 |
| 核函数 | 与分类SVM相同（线性/多项式/RBF/Sigmoid） |

#### 3.4 集成算法

**集成学习核心思想**：通过组合多个基学习器（Base Learner），获得比单一学习器更好的泛化性能。

##### 3.4.1 Bagging（Bootstrap Aggregating）

**核心原理**：自助采样（有放回抽样）生成多个训练子集，分别训练基学习器，最终聚合预测结果。

Bagging流程：
1. 从原始数据集D中自助采样（Bootstrap）生成M个子集D1, D2, ..., DM
2. 在每个子集上独立训练一个基学习器
3. 预测时：
   - 分类任务：多数投票（Majority Voting）
   - 回归任务：平均值（Averaging）

**代表算法：随机森林（Random Forest）**

| 维度 | 说明 |
| --- | --- |
| 基学习器 | 决策树 |
| 随机性来源 | 样本随机（Bootstrap）+ 特征随机（每棵树随机选特征子集） |
| 每棵树生长 | 通常不剪枝，充分生长 |
| 预测方式 | 分类：多数投票；回归：平均值 |

**随机森林关键特点**：

| 特点 | 说明 |
| --- | --- |
| 优点 | 抗过拟合能力强、能处理高维数据、能评估特征重要性、训练可并行、对缺失值不敏感 |
| 缺点 | 模型大（存储多棵树）、预测速度相对慢、黑盒模型（可解释性不如单棵决策树） |
| 超参数 | n_estimators（树的数量）、max_features（每棵树特征数）、max_depth（树深度） |
| 特征重要性 | 基于特征在树分裂中带来的不纯度减少总量 |

##### 3.4.2 Boosting

**核心原理**：串行训练基学习器，每个新学习器重点关注前面学习器分类错误的样本，通过加权组合提升整体性能。

**通用Boosting框架**：
1. 初始化样本权重（均匀分布）
2. For t = 1 to T:
   a. 根据当前样本权重训练基学习器h_t
   b. 计算h_t的误差/权重
   c. 更新样本权重（增加错分样本权重）
3. 组合所有基学习器得到最终模型

**四种Boosting算法对比（MECE）**：

| 算法 | 基学习器 | 权重更新方式 | 损失函数 | 特点 |
| --- | --- | --- | --- | --- |
| **Adaboost** | 弱分类器（如决策树桩） | 根据分类错误率调整样本权重和模型权重 | 指数损失 | 最早；对噪声敏感；简单有效 |
| **GBDT** | 决策树（CART回归树） | 拟合前面模型的负梯度（残差） | 可自定义（MSE/对数损失等） | 梯度提升框架；灵活强大；串行训练慢 |
| **Xgboost** | 决策树 | GBDT改进，加入正则化（L1+L2）、二阶泰勒展开 | 带正则化的损失函数 | 速度快（并行+缓存）、防止过拟合、支持缺失值自动处理、竞赛神器 |
| **LightGBM** | 决策树（叶子优先生长） | 基于直方图的梯度提升 | 带正则化的损失函数 | 速度最快（直方图算法+Leaf-wise）、内存占用小、大数据首选 |

**Adaboost详解**：
- 核心思想：聚焦错分样本，提升弱分类器为强分类器
- 每轮迭代：
  1. 根据样本权重训练弱分类器
  2. 计算分类误差率
  3. 计算该弱分类器的权重alpha
  4. 更新样本权重（错分样本权重增加）
  5. 归一化权重
- 最终预测：加权多数表决

**GBDT详解**：
- 核心思想：每棵新树拟合前面所有树的残差（负梯度）
- 训练过程：
  1. 初始化：所有样本预测值为目标均值
  2. 计算残差（负梯度）：ri = yi - F(xi)
  3. 用(xi, ri)训练回归树
  4. 更新模型：F(x) += learning_rate * h(x)
  5. 重复步骤2-4
- 特点：支持自定义损失函数；Shrinkage（学习率）防止过拟合

**Xgboost vs LightGBM核心差异**：

| 维度 | Xgboost | LightGBM |
| --- | --- | --- |
| 树的生长策略 | Level-wise（按层生长） | Leaf-wise（按叶子生长） |
| 分裂点查找 | 预排序+贪心 | 直方图算法 |
| 并行方式 | 特征并行 | 特征并行+数据并行 |
| 内存占用 | 较大 | 较小 |
| 训练速度 | 快 | 更快 |
| 处理大规模数据 | 好 | 更好 |
| 正则化 | L1 + L2 | L1 + L2 |
| 类别特征 | 需One-Hot编码 | 原生支持类别特征 |

##### 3.4.3 Stacking

**核心原理**：用多个不同类型的基学习器的预测结果作为输入，训练一个元学习器（Meta-learner）进行最终预测。

Stacking架构：

```
Layer 0（基学习器层）：
├── 学习器1：逻辑回归
├── 学习器2：SVM
├── 学习器3：随机森林
└── 学习器4：KNN
       ↓（各学习器的预测结果）
Layer 1（元学习器层）：
└── 元学习器：逻辑回归（或其他简单模型）
       ↓
    最终预测
```

**关键特点**：

| 特点 | 说明 |
| --- | --- |
| 优点 | 能融合不同类型模型的优势、灵活性高 |
| 缺点 | 训练复杂度高、容易过拟合（需交叉验证生成元特征） |
| 关键技巧 | 使用交叉验证生成元特征（防止信息泄露）、元学习器通常用简单模型 |
| 与Bagging区别 | Bagging用同类型学习器+投票；Stacking用不同类型学习器+元学习器 |
| 与Boosting区别 | Boosting串行训练、聚焦错分样本；Stacking并行训练、元学习器融合 |

##### 3.4.4 Voting

**核心原理**：多个基学习器独立预测，通过投票机制确定最终结果。

**投票方式（MECE）**：

| 投票方式 | 原理 | 适用场景 |
| --- | --- | --- |
| **硬投票（Hard Voting）** | 每个学习器输出类别标签，取多数类别 | 所有学习器都能输出确定类别 |
| **软投票（Soft Voting）** | 每个学习器输出类别概率，取概率平均值最大者 | 所有学习器都能输出概率（通常效果更好） |

**关键特点**：

| 特点 | 说明 |
| --- | --- |
| 优点 | 简单直观、易并行、无需额外训练元学习器 |
| 缺点 | 不如Stacking灵活、假设各学习器性能相近 |
| 与Stacking区别 | Voting直接投票；Stacking用元学习器学习最优组合权重 |

**四种集成方法对比总结**：

| 维度 | Bagging（随机森林） | Boosting | Stacking | Voting |
| --- | --- | --- | --- | --- |
| 基学习器关系 | 并行独立 | 串行依赖 | 并行独立 | 并行独立 |
| 基学习器类型 | 同类型 | 同类型 | 不同类型 | 不同类型 |
| 组合方式 | 投票/平均 | 加权组合 | 元学习器 | 投票 |
| 代表算法 | Random Forest | Xgboost, LightGBM | StackingClassifier | VotingClassifier |
| 主要目的 | 降低方差 | 降低偏差 | 融合优势 | 简单融合 |
| 训练速度 | 快（可并行） | 慢（串行） | 中 | 快（可并行） |
| 过拟合风险 | 低 | 中（需调参） | 高（需防泄露） | 低 |

### 6.5 常见学习误区与建议

| 误区 | 正确理解 | 建议 |
| --- | --- | --- |
| "数据挖掘=机器学习" | 数据挖掘是交叉学科，ML是核心技术之一 | 同时关注统计学、数据库、可视化 |
| "算法越复杂越好" | 简单模型往往更稳健、更易解释 | 从简单模型开始，根据效果决定是否升级 |
| "特征工程不重要" | 特征工程往往比算法选择更重要 | 投入足够时间做EDA和特征工程 |
| "集成一定比单模型好" | 集成可能过拟合、增加复杂度 | 先确保单模型调优到位，再考虑集成 |
| "LightGBM永远最快最好" | 不同场景适用不同算法 | 理解各算法优缺点，根据数据和场景选择 |
| "数据预处理可以自动化" | 预处理需要结合业务理解 | 不要完全依赖自动化工具，理解每一步的目的 |
| "只调参不做特征工程" | 好的特征比调参更重要 | 先做好特征工程，再精细化调参 |
